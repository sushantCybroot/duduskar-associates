# Duduskar & Associates Website

Premium law firm website built with Next.js App Router, Tailwind CSS, MongoDB, OTP admin login, contact inquiries, and a blog CMS.

## Getting Started

First, copy `.env.example` to `.env.local` and fill the required values:

```bash
MONGODB_URI=mongodb://...
NEXTAUTH_SECRET=replace-with-a-long-random-secret
NEXT_PUBLIC_SITE_URL=http://localhost:3000
GMAIL_EMAIL=your-gmail@gmail.com
GMAIL_PASSWORD=your-gmail-app-password
```

Then run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser.

## Main Routes

- `/` - law firm homepage
- `/blog` - public blog listing with search and category filters
- `/blog/[slug]` - article detail pages with dynamic metadata
- `/admin/login` - OTP login
- `/admin/dashboard` - protected blog dashboard
- `/admin/blogs/create` - create blog posts
- `/api/contact` - contact inquiry endpoint
- `/api/blogs` - blog CRUD API
- `/api/auth/send-otp` and `/api/auth/verify-otp` - admin OTP flow

In development, public blog pages can fall back to sample content if MongoDB is not configured. In production, MongoDB is required for blog pages, blog CRUD, OTP storage, contact inquiry storage, and admin dashboard data.

## MongoDB Production Setup

1. Create a MongoDB Atlas cluster and database, for example `duduskar-associates`.
2. Create a database user with read/write access.
3. Add your deployment IPs to Atlas Network Access. For Vercel, allow access from all IPs or use a private networking option.
4. Set `MONGODB_URI` in `.env.local` and production environment variables:

```bash
MONGODB_URI=mongodb+srv://<db-user>:<db-password>@<cluster-host>/duduskar-associates?retryWrites=true&w=majority
```

5. Verify deployment health at `/api/health`; `database` should report `connected`.

## Deployment

Deploy on Vercel and configure the same environment variables in the project settings. The sitemap and robots routes are generated by the App Router.
